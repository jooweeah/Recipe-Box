● Done. Here's how it all works:

  Parsing — parseAmount handles four formats:

  ┌─────────────┬───────────────────────┐
  │    Input    │        Parsed         │
  ├─────────────┼───────────────────────┤
  │ "2"         │ 2                     │
  ├─────────────┼───────────────────────┤
  │ "1.5"       │ 1.5                   │
  ├─────────────┼───────────────────────┤
  │ "1/2"       │ 0.5                   │
  ├─────────────┼───────────────────────┤
  │ "1 1/2"     │ 1.5                   │
  ├─────────────┼───────────────────────┤
  │ "a handful" │ null → returned as-is │
  └─────────────┴───────────────────────┘

  Formatting — rounds to 2 decimal places and strips trailing zeros, so 1.50 →
  "1.5", 2.00 → "2", 0.33 → "0.33".

  Scaler UI — only renders when the recipe has a servings value. Initialized to
  the base serving count on load. − button is clamped at 1. A "Reset" link
  appears only when the current value differs from the original, and snaps it
  back.

  Factor — scaledServings / servings is computed once and passed into
  scaleAmount(ing.amount, factor) for every ingredient. Non-numeric amounts
  (like "a handful") pass through unchanged.

> Write Firestore security rules so users can only read and write their own     
recipes. Show me where to paste them in the Firebase console.                   

● Here are the rules:

  rules_version = '2';
  service cloud.firestore {
    match /databases/{database}/documents {

      // Each user's profile document
      match /users/{userId} {
        allow read, write: if request.auth != null
                           && request.auth.uid == userId;

        // Each user's recipes subcollection
        match /recipes/{recipeId} {
          allow read, write: if request.auth != null
                             && request.auth.uid == userId;
        }
      }
    }
  }

  Where to paste them:

  1. Go to https://console.firebase.google.com
  2. Select your project
  3. In the left sidebar click Firestore Database
  4. Click the Rules tab at the top
  5. Replace everything in the editor with the rules above
  6. Click Publish

  ---
  What each rule does:

  ┌────────────────────────────────────┬───────────┬────────────────────────┐
  │                Path                │   Rule    │         Reason         │
  ├────────────────────────────────────┼───────────┼────────────────────────┤
  │                                    │           │ Protects the profile   │
  │ /users/{userId}                    │ auth.uid  │ document (display      │
  │                                    │ == userId │ name) — only the owner │
  │                                    │           │  can read or update it │
  ├────────────────────────────────────┼───────────┼────────────────────────┤
  │                                    │           │ Protects all recipes — │
  │ /users/{userId}/recipes/{recipeId} │ auth.uid  │  a user can only read, │
  │                                    │ == userId │  create, update, or    │
  │                                    │           │ delete their own       │
  └────────────────────────────────────┴───────────┴────────────────────────┘

  What these rules block:

  - Unauthenticated requests — request.auth != null rejects all reads/writes
  from signed-out users, including direct API calls
  - Cross-user access — user A cannot read or modify user B's recipes or
  profile, even if they know the document ID
  - Anything not explicitly matched — Firestore denies all paths not covered by
  a match rule by default, so no other collections are accessible
